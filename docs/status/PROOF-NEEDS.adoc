= Proof Requirements — systemet

== Proof tier

*Tier: T1 — Critical.* This repository's entire product is a theory and its
proofs. There is no code path to fall back on: an unproven systemet is a
wager, not a result, and the repo's own honesty files say so.

== Proof categories

[cols="1,3,1", options="header"]
|===
| Code | Meaning | Applies?

| *TP* | Typing proofs (type soundness, safety) | Yes — as DOM (the theory *is* a type system)
| *INV* | Invariant proofs | Yes — trope machines (ET-12, ET-13)
| *SEC* | Security proofs | No (no code)
| *CONC* | Concurrency proofs | No (no code)
| *ALG* | Algorithm proofs (termination, correctness) | Yes — normalizer totality + decidability (ET-1, ET-2)
| *ABI* | ABI/FFI proofs | *No — the ABI seam was removed; systemet has no FFI. The template's ABI-1..5/TP-1..2 rows are retired.*
| *DOM* | Domain-specific proofs | Yes — the ET ledger is the master list
|===

== The master list

The authoritative, citable requirements are the fifteen obligations
*ET-1..ET-15* in `docs/theory/OBLIGATIONS.adoc`, with:

* mechanization slots MECH-1 (L1 conversion) and MECH-2 (grade-algebra laws)
  targeted first — see PROOF-STATUS §Mechanization slots;
* ET-14 (TEA erasure) explicitly OPEN — a research target, not a milestone
  with a date;
* ET-10/ET-11 blocked on the L3 ADR
  (`docs/theory/06-l3-guarded-recursion.adoc`);
* prospective ET-16 (L0 lowering correctness) requires an ADR before it may
  be added.

Do not duplicate the statements here: one ledger, one place to drift.

== Dangerous patterns (still fully applicable)

Gated proof files must be free of prover escape hatches:

[cols="1,2", options="header"]
|===
| Prover | Banned

| Lean4 | `sorry`, `admit`, `axiom` (in gated files), `native_decide`
| Idris2 | `believe_me`, `assert_total`, `postulate`
| Agda | `postulate`, `TERMINATING`/`NON_TERMINATING` pragmas
| Coq | `Admitted`, `Axiom` (in gated files)
|===

Enforcement is mechanical: `scripts/scan-dangerous.sh` (comment-aware) and
the Lean4 transitive axiom audit (`Systemet/Audit.lean`). An escape hatch
that survives review anyway becomes a *named assumption* under the ET-15
registry rule — never a silent pass.
