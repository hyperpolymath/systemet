= TEST-NEEDS: systemet

This file previously described the rsr-template-repo's own test surface
(Zig integration tests, template-instantiation E2E, 21-workflow validation).
None of that is systemet's test surface. Honest restatement:

== What systemet's test surface actually is

systemet is a prose + proof repository. There is no application code, so
there are no unit/integration/E2E tests of behaviour. The verifiable
surfaces are:

[cols="2,2,1", options="header"]
|===
| Surface | Checker | Current count

| Proof checking (the real test suite)
| `scripts/check-proofs.sh` + per-prover MANIFESTs (lands with the
  mechanization PR; prover-absent = FAIL)
| 0 gated proofs

| Dangerous-pattern scan
| `scripts/scan-dangerous.sh` (comment-aware)
| pending (same PR)

| Repo shape (RSR)
| estate CI workflows + `scripts/validate-template.sh` (localized: enforces
  the IS-NOT — src/ must NOT exist)
| passing

| Doc integrity
| AsciiDoc-only-under-docs rule (`scripts/check-no-md-in-docs.sh`), link
  targets in docs/theory cross-references
| adoc rule enforced; link check manual

| Status-file drift
| PROOF-STATUS drift gate (recomputes counts from MANIFESTs; lands with the
  mechanization PR)
| pending
|===

== Explicitly not needed

* Unit/integration/E2E behaviour tests — no behaviour exists here.
* Benchmarks — nothing to benchmark; the template's `benches/` scaffolding
  is inert.
* Fuzzing — no parser or runtime in this repo.

If runnable code ever appears in this repository, that is first an IS-NOT
violation to resolve (the kernel belongs in anytype), not a reason to add
tests here.

== Residual template artefacts (kept, inert)

`tests/e2e.sh` (clean pass, no stack blocks enabled), `tests/aspect_tests.sh`,
`tests/e2e/template_instantiation_test.sh`, `benches/` — template scaffolding
retained because estate workflows reference their paths; none of them tests
systemet content. Removing them wholesale is a follow-up decision, not a
quiet edit.
